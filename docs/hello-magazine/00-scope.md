# 00. スコープ境界（重要）— マガジン(WordPress) と システム(別開発) の切り分け

本プロジェクト（このリポジトリ）は **ハローマガジンの記事コンテンツ = WordPress** のみを対象とする。
学校検索・口コミ・会員・課金などの**プラットフォーム本体は別システム（スクラッチ・AWS）**で、
**WordPress の対象外**。

出典: 「鈴木様 仕様・開発確認事項」シート（x.gd/cnckG）
- 「ソースコードの納品範囲：**wordpress側はテーマファイル一式、システム側はzipファイル一式**」
- 「記事コンテンツ（ハローマガジン）は更新しやすさ・SEO観点から **WordPressでの構築を希望**」

## 2つのサイト（別物）

| URL | 中身 | 基盤 | 本リポジトリ |
|-----|------|------|:--:|
| `https://magazine.hello-internationalschool.com/` | **ハローマガジン**（記事コンテンツ） | **WordPress** | ✅ 対象 |
| `https://hello-internationalschool.com/` | Hello! プラットフォーム本体 | 別システム（スクラッチ/AWS/zip） | ❌ 対象外 |

## ✅ WordPress（このリポジトリ）の対象範囲

ハローマガジンTOP（ワイヤー ⑲）と、その記事種別：
- YouTube LIVE 記事
- インタビュー 記事
- よくある質問（まとめ）
- ランキング（**編集記事として手動作成**。食べログ風の学校カード自動挿入は本体システム依存のため対象外/手動）
- エージェント比較（**編集記事**。一覧＋詳細）

付随して必要なもの（マガジンの範囲内）：
- 記事のタグ／話者(立場)／エージェント属性(カリキュラム・地域・価格帯)
- 記事の**手動多言語化**（Polylang）。※「学校記事の中国語翻訳・監修」等、記事の翻訳は WP 側で実施。

## ❌ WordPress の対象外（= 別システム側）

ワイヤー／開発メモに登場するが **WordPress では作らない**もの：
- 学校検索・**学校詳細ページ**・学校マスタ
- **口コミ**（投稿・表示・収集キャンペーン・800件一括投入・reviewsテーブル）
- **会員/認証**（LINE/Google/メール ログイン）、**トライアル/有料会員/Stripe 課金**、会員ランク
- **コミュニティ**（匿名トピック・コメント・通報/モデレーション）
- **受験ノート（学校比較）**
- **学校側/運営側 管理画面**（権限分け・申込/掲載管理）
- **UGC の自動翻訳**（DeepL/gTranslate・人力>自動のハイブリッド・監修キュー）
- マネタイズ（有料掲載プラン・支払フロー・約款・投稿ガイドライン 等の機能実装）

> 「開発メモ（桑澤記載）」(`1p-AO2dz`) は主に**後者（システム）**の仕様。
> 当初これをマガジン要件と取り違えていたため、本ドキュメントで切り分けを明確化した。
